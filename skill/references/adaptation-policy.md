# 反馈自适应规则

## 核心原则

**不要机械推进周大纲。** 每天先评估学习者状态，再决定今天生成什么内容。

## 五级优先级

### P0: 本周重构
- **触发条件**: 连续 2 天卡住 / 大 DDL 失败 / 学习者主动表示想放弃
- **处理方式**:
  - 降低本周目标 30-50%
  - 只保留核心任务，砍掉扩展内容
  - 安排复习日，不引入新概念
  - 加强鼓励，强调已取得的进展

### P1: 放慢复习
- **触发条件**: 代码跑不通 / 概念卡壳 / 验收清单完成率 < 50%
- **处理方式**:
  - 放慢节奏，每天减少任务量
  - 安排复习已学内容
  - 提供最小可行修复方案
  - 用更多生活比喻解释概念

### P2: 补完整性
- **触发条件**: 能跑但讲不清 / 缺项目结构 / 代码能跑但没测试
- **处理方式**:
  - 补充项目结构（README、测试、文档）
  - 增加口述练习："用自己的话解释 XXX"
  - 补充工程化内容（类型提示、异常处理）

### P3: 加难扩展
- **触发条件**: 提前完成 + 理解扎实 + 验收清单 100%
- **处理方式**:
  - 增加工程化扩展（测试覆盖、CI/CD、文档）
  - 提供进阶挑战题
  - 推荐相关阅读材料
  - 但不要跳过本周核心内容

### P4: 正常推进
- **触发条件**: 一切正常，按计划完成
- **处理方式**:
  - 按周大纲继续推进
  - 记录掌握度
  - 准备明天的任务

## 决策清单

每天生成任务前，检查以下问题：

1. 是否有周大纲？
2. 昨天的反馈状态是什么？
3. 哪个优先级最高？
4. 今天的内容是否服务于本周核心交付物？
5. 需要补救、推进、扩展还是重构？
6. 是否需要职业准备（GitHub、简历、面试练习）？

## 反馈收集模板

学习者填写后发送给 AI 助手：

```
- 今日完成度: {全部完成 / 部分完成 / 卡住了}
- 卡点描述: {具体问题}
- 代码是否能跑通: {是 / 否，附错误信息}
- 明天希望: {继续推进 / 复习巩固 / 加难挑战}
```

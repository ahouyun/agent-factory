在 Obsidian 中维护学习过程中的知识点，形成可复用的知识网络。

## 概念笔记模板

每个概念笔记包含：

```markdown
# {概念名称}

## 一句话解释
> {用最简单的话解释这个概念}

## 为什么重要
> {这个概念在 Agent 开发中的作用}

## 核心要点
- {要点 1}
- {要点 2}
- {要点 3}

## 代码示例
```python
# 最简示例代码
```

## 关联概念
- [[相关概念 1]]
- [[相关概念 2]]

## 首次出现
- 周次: Week {N}
- 日期: {Date}

## 掌握度
- [ ] 能解释这个概念
- [ ] 能写代码实现
- [ ] 能在面试中讲清楚
```

## 双链规则

1. **首次出现时链接** — 在每日任务中首次提到一个概念时，使用 `[[概念名]]` 链接
2. **不要过度链接** — 同一个概念在同一天只链接一次
3. **不要在代码块中链接** — 双链只在正文中使用
4. **统一命名** — 同一个概念始终使用相同的名称
